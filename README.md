How can I ensure that my code, program, or software is functional and secure?

As we laerned in this course, we can ensure that our software is functional through various testing styles, and in this particular case we used JUnit testing. This allowed us to test the software code to ensure that the software would act as expected when met with certain input/scenarios. We did this by implementing happy path testing, as well as boundary and negative testing top ensure the software wouldnt crash or expose vulnerabilities.

How do I interpret user needs and incorporate them into a program?

The most effective way to interpret user needs is to reference a client Requirements Document. In the begining stages of the software lifecycle, team members have put in time to make sure that the clients needs were properly documented, so it is best to reference this information. Having this information abailable it is important to design the software function according to the requirements, but most importantly write the tests to verify that the code meets the criteria. It m ay even be smart to write the tests according to the requirements first so that you build the code to meet those requirements.

How do I approach designing software?

Designing software starts with proper preperation and having access to all necessary information about the software, such as  functionalv  and non-functional requirements. OOP allows us to create software that is modular, maintainable, and clean. This means that any changes to the project or discovered flaws can be addressed efficiently and without having to overhaul entire lines of code. Classes or their corresponding service classes can be adressed individually based on needs.
